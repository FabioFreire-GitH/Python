Antes de expandir mais. Isso fecha o loop principal do sistema.
Se a sala controla exatamente quais subtipos podem aparecer, então você não deve abstrair demais.

Exemplo:

'Cripta': {
    'loot': ['moedas', 'joias', 'gemas']
}

Isso é ótimo porque restringe o universo do loot daquela sala.

Se você trocar para:

'loot': ['tesouro']

você perde esse controle fino.

Então o ideal é um modelo híbrido:

Camada 1 — Sala define os subtipos permitidos

Exemplo:

'loot': ['moedas', 'joias', 'arma', 'pocao', 'nada']

Perfeito.

Camada 2 — gerar_loot interpreta

Ele analisa:

Se for:

moedas
joias
gemas
artefatos
objetos valiosos

→ chama:

gerar_tesouro([tipo], bonus)

Se for:

arma
armadura
cura

→ chama:

gerar_item([tipo], bonus)

Então o ajuste real é:

Hoje seu gerar_item() sorteia internamente:

lista_tipo = ['cura','arma','armadura']

Troque para aceitar filtro:

Exemplo conceitual:

gerar_item(tipo_item, prob)

Assim:

gerar_item('arma')
gerar_item('armadura')
gerar_item('cura')

Mesmo raciocínio para tesouro:

gerar_tesouro(['joias'], prob)

ou:

gerar_tesouro(['moedas','joias'], prob)

Estrutura ideal para você:

Sala escolhe exatamente o que pode dropar
↓

↓

↓
